# OpenClaw SRE 自我调控系统

> 基于钱学森《工程控制论》的 AI 任务调度稳定性控制系统

## ⚠️ 重要声明

**这是一个经验系统，不是理论证明的控制论实现。**

系统使用钱学森工程控制论的**概念和术语**，但：
- K(n) 描述函数是**经验公式**，无严格数学证明
- LAUNCH/fuse 阈值是**工程经验值**，非理论推导
- 控制效果依赖于 OpenClaw 实际任务特性

**使用前请阅读本文档，了解系统能力边界。**

## 核心指标

| 指标 | 公式 | 目标 | 说明 |
|------|------|------|------|
| n (负载率) | q / max_c | < 0.5 | 核心控制目标 |
| H (健康度) | 0.6×load_health + 0.4×quality_health | > 0.5 | 系统总体健康 |
| L (负载度) | min(n/2, 1) | < 0.5 | 负载水平 |
| q (队列) | 实际队列长度 | < 30 | 直观指标 |

## 五态分类

| 状态 | 数学条件 | 控制策略 |
|------|---------|---------|
| stable | H>0.5 且 L<0.5 | 常规运行 |
| warning | H<0.5 | 深度分析 |
| overload | L>0.5 且 H>0.4 | SAGI 降增益 |
| diverging | H<0.4 且 L>0.5 | 熔断 |
| drifting | 其他 | 提高采样频率 |

## 萨奇图 (SAGI) 控制

### 三区参数

```python
SAGI_ZONES = [
    (0.0,  0.5,  "safe",       0,   0,  1.0,  1.0,  42, 38),
    (0.5,  0.8,  "elevated",  -8,  -8,  0.6,  0.6,  45, 38),
    (0.8,  99.0, "critical",  15,  12,  2.0, 1.0, 50, 45),
]
# 元组: (n_min, n_max, zone_name, don, doff, maxcf, gf, base_on, base_off)
```

### maxcf 的作用

| Zone | maxcf | 数学效果 | 控制意图 |
|------|-------|---------|---------|
| safe | 1.0 | 无变化 | 不控制 |
| elevated | 0.6 | n×1.67 | 略微降低容量 |
| critical | 2.0 | n×0.5 | **提高容量**，降低负载率 |

> ⚠️ **关键**：critical 区 maxcf > 1 是为了**提高容量**而非降低，这是基于 n = q/max_c 的数学关系。

### 控制动作

| Zone | on 增量 | off 增量 | 效果 |
|------|---------|---------|------|
| safe | +0 | +0 | 不调整 |
| elevated | -8 | -8 | 降低派发阈值 |
| critical | +15 | +12 | 提高派发阈值，更保守 |

## 发射段模式 (LAUNCH)

### 进入条件

```python
enter_launch = delta_H > 0.12 or delta_n > 0.3 or is_drifting
# 需要连续 2 周期满足（滞环）
```

### 退出条件

| 条件 | 说明 |
|------|------|
| base_exit | delta_H<0.06 且 delta_n<0.15 且 H>0.5 且 n<0.4 |
| q_rate 退出 | q 持续下降 (q_rate < -0.1) |
| 超时退出 | 600 秒或 suppressed_cycles ≥ 10 |

### 冷却机制

LAUNCH timeout 退出后，60 秒内不会重新进入 LAUNCH 模式。

## 熔断器 (fuse)

### 触发条件

```python
fuse_triggered = (H < 0.25 or L > 0.7 or n > 1.5)
```

### 熔断动作

```python
SPAWN_ON = max(5, SPAWN_ON - 15)   # 降低派发
MAX_C = max(1, MAX_C × 0.25)        # 降低容量
```

### 恢复条件

```python
fuse_recovery = consecutive_safe >= 3 and H > 0.5 and L < 0.5
```

## 描述函数 K(n)

### 数学定义

```python
K(n) = K_max × (1 - (n/n_max)^p)  # n < n_max 时
K(n) = K_sat                          # n ≥ n_max 时
```

### 物理意义

K(n) 代表 Agent 在负载率 n 下的**实际能力系数**：

| n | K(n) | 能力 |
|---|------|------|
| 0.0 | 0.95 | 100% |
| 0.5 | 0.75 | 79% |
| 1.0 | 0.25 | 26% |
| >1.0 | 0.10 | 10% |

> ⚠️ 这是**经验公式**，没有严格的控制论证明。

## 已知局限性

### 1. 理论层面

| 问题 | 影响 | 缓解 |
|------|------|------|
| K(n) 无稳定性证明 | 无法保证控制效果 | 监控实际效果 |
| 阈值是经验值 | 最优性未知 | 可调参数 |
| 双回路不协调 | 可能控制冲突 | 分离监控/控制 |

### 2. 工程层面

| 问题 | 影响 | 缓解 |
|------|------|------|
| 单点故障 | 文件损坏影响大 | 备份机制 |
| 非实时 | 控制有滞后 | 高频采样 |
| 无集成测试 | 难以验证修复 | 单元测试覆盖 |

## 调参指南

### 何时需要调参

- 系统持续在 LAUNCH 模式无法退出
- fuse 频繁触发
- n 持续 > 0.5
- 任务成功率持续 < 30%

### 关键参数

| 参数 | 作用 | 安全范围 |
|------|------|---------|
| LAUNCH_TIMEOUT_SECS | 超时退出 | 300-900 |
| CHANGE_THRESHOLD | 变化检测灵敏度 | 0.1-0.2 |
| SUPPRESSION_TIMEOUT | suppressed_cycles 上限 | 5-20 |
| CRITICAL_SPAWN_ON_INCREASE | critical 区派发增量 | 10-20 |

### 调参原则

1. **小步调整**：每次修改不超过 20%
2. **观察效果**：至少 10 个周期后再判断
3. **记录变更**：每次修改写入日志
4. **回滚准备**：修改前备份当前状态

## 故障排查

### 系统进入 LAUNCH 无法退出

**检查**：
1. H 是否 > 0.5
2. n 是否 < 0.4
3. delta_H 是否持续 > 0.06

**可能原因**：
- EWMA 快慢滤波尚未收敛（等待或重启 ewma_state）
- 参数设置过于保守

### fuse 频繁触发

**检查**：
1. q 是否持续 > 20
2. H 是否 < 0.25

**可能原因**：
- 系统负载过高
- max_c 设置过低

### LAUNCH 反复进入/退出

**检查**：
1. 是否 timeout 触发
2. delta_H 是否持续在阈值附近

**可能原因**：
- 正常收敛过程（等待）
- 冷却期过短（增加 cooldown 时间）
