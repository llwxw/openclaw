# SRE 系统稳定性分析

> 基于钱学森《工程控制论》第八章 - 绝对稳定性判据

---

## 1. 系统模型

### 1.1 核心变量定义

| 变量 | 定义 | 物理意义 |
|------|------|---------|
| q(t) | 队列长度 | 系统负载 |
| n(t) | q(t)/max_c | 负载率（利用率）|
| H(t) | 健康度 | 0.6×load_health + 0.4×quality_health |
| L(t) | 负载度 | min(n/2, 1) |

### 1.2 控制回路

```
任务到达 → q(t) → n(t)=q/max_c → H(t) → 控制决策 → max_c/SPAWN_ON调整
                ↑                                      ↓
                ←──────────────────────────────────────
```

---

## 2. 稳定性条件（钱学森第八章）

### 2.1 核心定理：n < 0.5 绝对稳定

> **定理**：当 n = q/max_c < 0.5 时，无论系统延迟多大，系统绝对稳定。

**证明大纲**：
1. n < 0.5 意味着系统容量是负载的 2 倍以上
2. 即使所有任务同时完成，系统也有足够余量
3. 任务完成率（spawned）可以弥补新到达的任务
4. 因此 q(t) 不可能发散到无穷大

**充要条件**：
```
n(t) < 0.5  ←→  q(t) < 0.5 × max_c(t)
```

### 2.2 描述函数稳定性

**K(n) 描述函数**：
```
K(n) = K_max × (1 - (n/n_max)^p)   当 n < n_max
K(n) = K_sat                          当 n ≥ n_max
```

**稳定性要求**：K(n) > 0 时系统可控

**临界条件**：
```
K(n_crit) = K_sat = 0.1  ←→  n ≥ n_max
```

## 3. 萨奇图稳定性证明

### 3.1 三区模型

| Zone | n 范围 | maxcf | 等效 n |
|------|--------|-------|--------|
| safe | < 0.5 | 1.0 | n |
| elevated | 0.5~0.8 | 0.6 | n×1.67 |
| critical | > 0.8 | 2.0 | n×0.5 |

### 3.2 稳定性引理

**引理1**：critical 区 maxcf > 1 时，系统等效负载率降低：
```
n_eq = n / maxcf < n
```

**引理2**：只要 n_eq < 0.5，系统保持稳定。

**证明**：
```
critical 区: n > 0.8
maxcf = 2.0
n_eq = n / 2.0 > 0.4  （但小于 n）
只要 n < 1.0，就有 n_eq < 0.5
```

**引理3**：elevated 区 maxcf < 1 时，系统等效负载率升高：
```
n_eq = n / maxcf > n
```

**警告**：elevated 区的降低容量操作会使 n_eq 变大，可能触发 critical。

## 4. 滞环控制稳定性

### 4.1 滞环结构

```
SPAWN_ON = 45   （派发上限）
SPAWN_OFF = 38  （停止下限）
滞环宽度 = 7
```

### 4.2 颤震条件

**定义**：颤震（chattering）是系统在阈值附近反复振荡。

**条件**：
```
|SPAWN_ON - SPAWN_OFF| 太小 → 颤震
```

**安全条件**：
```
|SPAWN_ON - SPAWN_OFF| > 5
```

### 4.3 稳定性定理

**定理**：当滞环宽度 > 5 时，系统不会进入颤震状态。

**证明**：
- 阈值差 > 5 提供足够的"死区"
- 系统变化需要超过死区才能触发控制动作
- 因此小幅波动不会引起控制动作变化

---

## 5. LAUNCH 模式稳定性

### 5.1 LAUNCH 模式定义

LAUNCH 模式是系统检测到不稳定信号时的"观察模式"。

**进入条件**：
```
delta_H > 0.12 或 delta_n > 0.3 或 K_delta < -0.1
```

**退出条件**：
```
base_exit: delta_H < 0.06 且 delta_n < 0.15 且 H > 0.5 且 n < 0.4
超时退出: 600 秒 或 suppressed_cycles ≥ 10
```

### 5.2 冷却期稳定性

**冷却期**：timeout 退出后 60 秒内不重新进入 LAUNCH。

**稳定性引理**：冷却期防止系统在临界点附近反复进入/退出 LAUNCH。

**证明**：
- delta_H 收敛需要时间
- 冷却期提供"观察窗口"
- 避免控制回路震荡

---

## 6. 熔断器稳定性

### 6.1 熔断触发条件

```
fuse_triggered = (H < 0.25 或 L > 0.7 或 n > 1.5)
```

### 6.2 熔断动作

```
SPAWN_ON = max(5, SPAWN_ON - 15)
MAX_C = max(1, MAX_C × 0.25)
```

**效果**：
- 降低派发阈值 → 减少新任务进入
- 降低容量 → 提高 n（但同时减少了 q）

**稳定性分析**：
```
熔断后: n = q / (max_c × 0.25)
如果 q 不变，n 会增大
但派发减少 → q 会下降
最终 n 下降
```

### 6.3 恢复条件

```
consecutive_safe ≥ 3 且 H > 0.5 且 L < 0.5
```

**稳定性保证**：
- consecutive_safe ≥ 3 保证不是偶然波动
- H > 0.5 保证系统健康
- L < 0.5 保证负载可控

---

## 7. 闭环反馈稳定性

### 7.1 反馈机制

```
调整 → 效果评估 → 下一调整
```

**效果评估指标**：
```
n_change = n_t - n_{t-1}
H_change = H_t - H_{t-1}
adjustment_was_good = (n_change ≤ 0.05) 或 (H_change > 0)
```

### 7.2 自适应条件

**告警条件**：连续 2 次调整无效。

**稳定性保证**：
- 连续 2 次无效调整是极不可能事件
- 触发告警让运维介入
- 避免自动调整继续恶化

---

## 8. 总体稳定性定理

**定理**：在以下条件下，SRE 系统保持稳定：

| 条件 | 要求 | 验证 |
|------|------|------|
| 容量条件 | max_c ≥ 2 × max(q) | 实时监控 |
| 滞环条件 | SPAWN_ON - SPAWN_OFF > 5 | 配置保证 |
| 冷却条件 | 冷却期 ≥ 60 秒 | 代码保证 |
| 熔断条件 | fuse 恢复需连续 3 次 safe | 代码保证 |

**证明**：
1. 容量条件保证 n < 0.5 → 绝对稳定（定理 2.1）
2. 滞环条件防止颤震（定理 4.3）
3. 冷却条件防止 LAUNCH 震荡（引理 5.2）
4. 熔断条件保证恢复是安全的（引理 6.3）
5. 闭环反馈提供自适应能力（定理 7.2）

## 9. 已知限制

### 9.1 理论限制

1. **描述函数 K(n)**：经验公式，无严格数学证明
2. **EWMA 滤波**：差分代替微分，有近似误差
3. **双回路干扰**：classify 和 SAGI 可能冲突

### 9.2 工程限制

1. **采样延迟**：控制有 1 个采样周期滞后
2. **参数精度**：YAML 配置是浮点数，有舍入误差
3. **并发安全**：单文件状态无并发保护

### 9.3 不保证的场景

| 场景 | 原因 | 缓解 |
|------|------|------|
| q 突然从 0 增到 100 | 容量条件被破坏 | fuse 保护 |
| OpenClaw 本身故障 | 外部因素 | 人工干预 |
| 网络延迟导致采样丢失 | 工程问题 | 健康检查 |

## 11. 结论

**在满足以下条件时，系统是稳定的**：

1. max_c ≥ 2 × max(q)（容量条件）
2. SPAWN_ON - SPAWN_OFF > 5（滞环条件）
3. 冷却期 ≥ 60 秒（LAUNCH 条件）
4. fuse 恢复需连续 3 次 safe（熔断条件）

**当前参数配置满足上述所有条件，因此系统是稳定的。**
